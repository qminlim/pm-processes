📌 Exam Tip — Key Rule
Planning is the LARGEST process group with 24 processes. If an exam question asks "what should the PM do first?", the answer is almost always related to planning before acting. Only exceptions: if there's an approved change request (execute it) or a crisis (monitor/control).

4 Baselines & 14 Subsidiary Plans
Inside the Project Management Plan
▾
4 Baselines (SSCP)
Scope · Schedule · Cost · Performance Measurement
  • Scope Baseline = Scope Statement + WBS + WBS Dictionary
  • Schedule Baseline = approved project schedule
  • Cost Baseline = S-curve (cost + contingency reserves)
  • Project Budget = Cost Baseline + Management Reserves
    • 📌 Exam Tip
    Baselines can ONLY be changed through an approved Change Request via the Change Control Board (CCB).

    Change Request Process
    Integrated Change Control flow
    ▾
  • Stakeholder identifies need for change
  • Written change request submitted to PM
  • PM assesses impact on all constraints
  • Submitted to Change Control Board (CCB)
  • CCB approves or rejects
  • If approved → update PM Plan and manage to new plan
  • If rejected → develop new change request (repeat)
    • 📌 Exam Tip
    ANY stakeholder can submit a change request. All requests should be in written form. The PM never approves changes unilaterally — that's the CCB's role.

    Develop Project Management Plan
    Planning · Integration
    ▾
  • Defines, prepares, and coordinates all plan components
  • Comprehensive document outlining basis of all project work
  • Contains 4 Baselines + 14 Subsidiary plans
  • Only changed via approved Change Request from CCB
    • 14 Subsidiary Plans (in PM Plan)
    Scope · Requirements · Schedule · Cost · Quality · Resource · Communication · Risk · Procurement · Stakeholder · Change · Configuration · Project Life Cycle · Development Approach
    📌 Exam Tip
    The PM Plan is approved by: PM, Sponsor, Functional Manager, Program Manager, or (rarely) Senior Management. It may be summary-level OR detailed.

    Plan Scope Management
    Planning · Scope
    ▾
  • Creates scope management plan — how scope will be defined, validated, and controlled
  • Product Scope — features/functions of a product, service, or result
  • Project Scope — work needed to deliver the product
    • Outputs
    Scope Management Plan — how scope will be defined, developed, monitored, controlled, verified

    Requirements Management Plan — how requirements will be analyzed, documented, and managed (+ traceability structure)
    📌 Exam Tip
    Gold Plating = doing extra work NOT in scope (avoid!). Scope Creep = unauthorized additions to scope (prevent!). Both are project risks.

    Collect Requirements
    Planning · Scope
    ▾
  • Determining, documenting, and managing stakeholder needs and requirements
  • Critical to project success — requirements drive schedule, budget, risk, quality, resources
  • Data Gathering — Benchmarking, document analysis
  • Data Representation — Mind Mapping (brainstorm ideas), Affinity Diagrams (group large ideas)
  • Context Diagrams — Shows how systems and people interact
  • Prototypes — Working model for stakeholder feedback
    • Outputs
    Requirements Documentation — how each req is performed and why important

    Requirements Traceability Matrix (RTM) — links requirements back to source; tracks who, why, description, status
    📌 Exam Tip
    The RTM is used to manage scope changes. It tracks the original stakeholder, reason, description, and current status (completed, in-progress, delayed, cancelled).

    Define Scope
    Planning · Scope
    ▾
  • Developing a detailed description of the project and product
  • Key tool: Product Analysis — product breakdown, system analysis, system requirements
    • Project Scope Statement Includes
    Product description · Project goals · Identified risks · Acceptance criteria · Constraints/exclusions
    📌 Exam Tip
    More detail in the scope statement = less project risk and less scope creep. Less detail = more risk. Always push for detailed scope statements on the exam.

    Create WBS (Work Breakdown Structure)
    Planning · Scope
    ▾
  • Subdividing project deliverables into smaller, manageable components
  • If it's NOT in the WBS, it's NOT part of the project
  • Created by PM, SMEs, and Project Team (great team-building tool)
  • Each node must have a unique identifying number — no gaps, no overlaps
  • A deliverable-oriented hierarchical decomposition
    • Scope Baseline (3 Components)
    1. Project Scope Statement
    2. WBS
    3. WBS Dictionary
    WBS Dictionary Contains
    Team member assigned · Time estimate · Cost estimate · Account info · Work Package ID · Quality requirements · Contract info · Scheduled Milestones
    📌 Exam Tip
    Work Package = lowest level of WBS (deliverable-based). Activity = work done to complete a work package (schedule-focused). One work package can have many activities.

    Plan Schedule Management
    Planning · Schedule
    ▾
  • Establishes policies, procedures, documentation for schedule
  • Output: Schedule Management Plan — levels of accuracy, rules of performance measurement, reporting formats, release/iteration length, project schedule model development
    • 🔢
    Define Activities
    Planning · Schedule
    ▾
  • Identifying and documenting specific actions to produce deliverables
  • Decomposes work packages into schedule activities
    • Key Tool — Rolling Wave Planning
    Progressive elaboration technique. Near-term work packages defined in detail; long-term work packages get placeholders updated as more info becomes available.
    Outputs
    Activity List · Activity Attributes · Milestone List · Change Requests
    🔗
    Sequence Activities
    Planning · Schedule
    ▾
  • Identifying relationships among project activities
  • Key tool: PDM (Precedence Diagramming Method) — graphical representation of project flow
    • 4 Dependency Relationships (PDM)
    FS (most common) — Finish-to-Start
    FF — Finish-to-Finish
    SS — Start-to-Start
    SF — Start-to-Finish (rarest)
    Dependency Types
    Mandatory (Hard Logic) — must be done in order (physical limitation)
    Discretionary (Soft Logic) — can work in parallel
    External — outside team control
    Internal — within team control
    Leads & Lags
    Lead — Advance a successor (start earlier than predecessor finishes)
    Lag — Delay a successor (wait before starting)
    📌 Exam Tip
    Finish-to-Start (FS) is the MOST commonly used dependency. On the exam, if no relationship type is specified, assume FS.
    ⏱️
    Estimate Activity Durations
    Planning · Schedule
    ▾
  • Estimating work periods needed to complete individual activities
  • Should be calculated by the person most familiar with the work
    • Estimating Techniques
    Analogous (Top-Down) — historical data; least accurate, least expensive
    Parametric — statistical relationship (e.g., sqft, lines of code)
    Three-Point (PERT) — (O + 4M + P) / 6
    Bottom-Up — highest accuracy, most time-consuming
    Reserve Analysis — contingency/buffer for risk
    PERT Beta Distribution Formula
    (Optimistic + 4×Most Likely + Pessimistic) ÷ 6

    Example: O=8, P=14, M=10 → (8 + 40 + 14) / 6 = 10.33 days

    Triangle Distribution: (O + M + P) ÷ 3
    Standard Deviation: (P − O) ÷ 6
    📌 Exam Tip
    PERT uses weighted average — Most Likely gets 4× weight. Bottom-Up is the most accurate but labor-intensive. Analogous is quickest but least accurate. Match technique to information available.
    📊
    Develop Schedule
    Planning · Schedule
    ▾
  • Analyzing sequences, durations, resources, constraints to create schedule model
  • Generates planned dates for completing activities
    • Key Schedule Tools
    Critical Path Method (CPM) — calculates ES, EF, LS, LF; determines float/slack
    Critical Chain Method — emphasizes resource constraints
    Resource Optimization — flattens schedule when over-allocated
    Schedule Compression:
    · Crashing — add resources; ALWAYS adds cost, may add risk
    · Fast Tracking — parallel activities; may NOT add cost, INCREASES risk of rework
    📌 Exam Tip
    If asked how to shorten schedule WITHOUT adding cost → Fast Tracking. If budget is not a concern → Crashing. Critical Path = longest path through the network (activities with ZERO float).
    💰
    Plan Cost Management + Estimate Costs
    Planning · Cost
    ▾
    Cost Types
    Fixed — stays same (bulldozer rental) | Variable — changes (labor, fuel)
    Direct — billed directly to project | Indirect — shared across projects (manager salary)
    Sunk Costs — already spent; irrelevant to future decisions ("spilt milk")
    Estimate Types (Accuracy Ranges)
    Definitive: −5% to +10%
    Budget: −10% to +25%
    Rough Order of Magnitude (ROM): −25% to +75%
    Cost of Quality (COQ)
    Conformance: Prevention costs + Appraisal costs (doing it right)
    Non-Conformance: Internal failure (rework, scrap) + External failure (warranty, lost business)
    📌 Exam Tip
    Value Engineering = finding a less costly way to achieve the same scope/goal. Always ignore sunk costs when making decisions — they cannot be recovered.
    🏦
    Determine Budget
    Planning · Cost
    ▾
  • Aggregating estimated costs of activities to establish cost baseline
  • Cost Baseline is displayed as an S-Curve graph
    • Reserve Hierarchy
    Activities → Work Packages → Control Accounts

    Contingency Reserve — PM controls; for known/unknown risks
    Management Reserve — Management controls; for unknown/unknown risks

    Cost Baseline = all activity costs + contingency reserves
    Project Budget = Cost Baseline + Management Reserves
    📌 Exam Tip
    The Cost Baseline is the S-curve. Project Budget is ABOVE the baseline. Contingency reserves are in the baseline; management reserves are above it.
    ✅
    Plan Quality Management
    Planning · Quality
    ▾
  • Identifying quality requirements/standards and documenting compliance approach
  • Always cost more if customers find defects — prevent defects from reaching customers
  • Build quality into planning and design, not after
    • Data Analysis Tools
    Cost-Benefit Analysis — benefits must outweigh costs
    Cost of Quality (COQ) — all costs ensuring quality over product life
    Data Representation Tools
    Logical Data Model · Matrix Diagram · Mind Mapping · Flowcharts
    📌 Exam Tip
    Quality Metrics = specifications on HOW quality will be measured (e.g., "errors per line of code"). These are used during Control Quality.
    👤
    Plan Resource Management
    Planning · Resource
    ▾
  • Describing how to estimate, acquire, manage, and use team and physical resources
  • Team resources — people building deliverables
  • Physical resources — supplies, materials, services, facilities, equipment
    • Org Chart Types (Data Representation)
    Hierarchical — graphic top-down format (like WBS)
    Matrix-Based (RACI) — Responsibility Assignment Matrix
    Text-Oriented — detailed role descriptions
    RACI Chart
    Responsible · Accountable · Consulted · Informed
    📌 Exam Tip
    The Team Charter outlines acceptable behavior, rules of conduct for meetings, decision-making, and one-on-one conversations. Resource Management Plan guides all 6 resource management processes.

    Plan Communications Management
    Planning · Communications
    ▾
  • Developing appropriate approach based on stakeholder information needs
    • Communications Channels Formula
    Channels = n(n−1) / 2

    Example: 10 stakeholders → 10(10−1)/2 = 45 channels
    4 people → 4(4−1)/2 = 6 channels
    Communication Methods
    Push — email blast (one-way, sender initiates)
    Pull — download/portal (receiver retrieves)
    Interactive — joint discussions (two-way, real-time)

    Informal Written · Formal Written (contracts) · Informal Verbal · Formal Verbal (presentations)
    Communication Model Components
    Sender → Encoder → Medium → Decoder → Receiver
    Noise · Paralingual (pitch/tone) · Nonverbal · Blockers
    📌 Exam Tip
    Communications formula is a top exam topic. Adding one person to a 10-person team changes channels from 45 to 55 (10 new channels). Lack of communication is the #1 cause of project failure.

    Risk Management Planning & Identification
    Planning · Risk
    ▾
  • Individual Risk — uncertain event with positive/negative impact on parts of the project
  • Overall Project Risk — sum of all individual risks + other sources of uncertainty
  • Negative risks = threats; Positive risks = opportunities
  • Risk Management = proactive approach, done early
  • All personnel should identify risks; risk changes daily
    • Risk Management Plan (Roadmap to 6 Risk Processes)
    Covers: risk categorization, qualitative/quantitative analysis approach, response planning, implementation, monitoring, ongoing activities, roles/responsibilities, stakeholder risk appetite, RBS (Risk Breakdown Structure)
    Identify Risks — Key Tools
    SWOT Analysis · Root Cause Analysis · Assumption/Constraint Analysis · Document Analysis · Prompt Lists
    Risk Register Contains
    List of all risks · Potential responses · Root causes · Risk categories
    📉
    Qualitative & Quantitative Risk Analysis
    Planning · Risk
    ▾
  • Qualitative — prioritizes risks by probability and impact; creates ranking; performed throughout project
  • Quantitative — numerically analyzes effect of risks on objectives; assigns monetary/numeric value
    • Qualitative Tools
    Probability & Impact Matrix (P×I = Score) · Risk Data Quality Assessment · Risk Categorization · Bubble Charts
    Quantitative Tools
    Monte Carlo Simulation · Sensitivity Analysis · Tornado Chart · Decision Tree · EMV (Expected Monetary Value)
    EMV Formula
    EMV = Probability × Risk Cost
    Total = Initial Cost + EMV

    Example: New house: $1M initial, $400K risk @ 25% → EMV = $100K → Total = $1.1M
    📌 Exam Tip
    Always do Qualitative BEFORE Quantitative. Qualitative filters which risks need deeper analysis. Quantitative requires specialized software and expertise.
    🛡️
    Plan Risk Responses
    Planning · Risk
    ▾
    Threat Strategies (EEAMT)
    Escalate — outside PM authority
    Avoid — eliminate the risk entirely
    Transfer — shift ownership to 3rd party (insurance, contracts)
    Mitigate — reduce probability/impact
    Accept — passive (do nothing) or active (contingency plan)
    Opportunity Strategies (EEESHA)
    Escalate · Exploit (remove all uncertainty) · Share (3rd party) · Enhance (increase probability) · Accept
    📌 Exam Tip
    Escalate (both threats & opportunities) is used when risks are outside PM's authority. Transfer does NOT eliminate risk — it moves ownership. Mitigate reduces but doesn't eliminate.
    🤝
    Plan Procurement Management
    Planning · Procurement
    ▾
  • Determines whether to obtain goods/services externally and how/when
    • Contract Types
    Fixed Price (Lump Sum) — all risk with seller; well-defined scope
    · FFP (Firm Fixed Price) — price cannot change
    · FPIF (Fixed Price Incentive Fee) — fee for meeting target
    · FP-EPA — adjusted for economic conditions

    Cost-Reimbursable — all risk with buyer
    · CPFF — pay expenses + fixed fee
    · CPIF — expenses + fee if target met
    · CPAF — expenses + award based on satisfaction

    Time & Material (T&M) — buyer takes all risk; use only when scope is high-level
    Bid Documents
    RFI — Request for Information
    IFB — Invitation for Bid
    RFP — Request for Proposal
    RFQ — Request for Quote
    📌 Exam Tip
    Fixed Price = seller at risk. Cost-Reimbursable = buyer at risk. T&M = highest buyer risk. Use Fixed Price when scope is well-defined. Use T&M when scope is unclear.

    Plan Stakeholder Engagement
    Planning · Stakeholder
    ▾
  • Developing methods to involve project stakeholders based on needs/expectations/interests
  • Creates an actionable plan for effective stakeholder interaction
    • Stakeholder Engagement Assessment Matrix — 5 Levels
    Unaware → Resistant → Neutral → Supportive → Leading

    Matrix shows "Current" vs "Desired" engagement level for each stakeholder
    📌 Exam Tip
    Goal: move stakeholders toward "Supportive" or "Leading". The matrix identifies gaps between current and desired engagement for each person.

    Direct and Manage Project Work
    Executing · Integration
    ▾
  • Performing the work defined in the PM Plan
  • Managing people, improving processes, requesting changes, implementing approved changes
  • Summary of all other executing processes
    • Key Outputs
    Deliverables — any product, service, or result required
    Work Performance Data — raw status info (cost/duration tracking)
    Issue Log — all problems encountered; assigned, prioritized, addressed
    Change Requests:
    · Corrective Action — fixing past errors (realign performance)
    · Preventive Action — fixing future errors (prevent misalignment)
    · Defect Repair — modify nonconforming product
    📌 Exam Tip
    Work Performance Data (raw) → becomes Work Performance Information (analyzed) → becomes Work Performance Reports (distributed). Know this flow!

    Manage Project Knowledge
    Executing · Integration
    ▾
  • Using existing knowledge and creating new knowledge for organizational learning
  • Explicit Knowledge — formally documented (data, records, documents)
  • Tacit Knowledge — in people's heads (experience, thinking, beliefs)
  • Output: Lessons Learned Register — gathered throughout the project, not just at end
    • 📌 Exam Tip
    Lessons Learned are collected throughout the project lifecycle, not only at the end. Updated whenever new knowledge is discovered by any stakeholder.

    Manage Quality
    Executing · Quality
    ▾
  • Translating the Quality Management Plan into executable quality activities
  • Also called Quality Assurance
  • Confirms quality processes are meeting quality objectives
    • Quality Charts (Data Representation)
    Cause & Effect (Fishbone/Ishikawa) — identifies root causes of defects
    Flowcharts — graphical process representation
    Histograms/Pareto Diagrams — 80/20 rule; bar charts of defect frequency
    Scatter Diagrams — trends between variables
    Control Charts — is process "in control"? Rule of 7
    📌 Exam Tip
    Pareto Principle (80/20) — 80% of problems come from 20% of causes. Focus on the vital few. Rule of Seven on control charts = 7 consecutive points on one side of mean = out of control.

    Acquire Resources
    Executing · Resource
    ▾
  • Getting staff and physical resources needed to build deliverables
  • Done continuously throughout the project
  • Both internal and external resources
  • Pre-Assignment — team members selected in advance of the project
  • Virtual Teams — geographically distributed, work from home, different shifts
  • Multi-Criteria Decision Analysis — availability, cost, experience, ability, skills, attitude

    Develop Team
    Executing · Resource
    ▾
  • Improving abilities, communication, and overall team atmosphere
  • Co-Location (Tight Matrix) — war room; entire team in one physical location
    • Tuckman's Ladder (5 Stages)
    Forming → Storming → Norming → Performing → Adjourning
    Motivation Theories
    Maslow's Hierarchy: Physiological → Safety → Social → Esteem → Self-Actualization

    Herzberg's Theory: Hygiene factors (prevent dissatisfaction) vs. Motivators (drive satisfaction)

    McGregor's X/Y: X = distrusted/micromanaged; Y = self-motivated/proactive

    Theory Z: Loyalty through employee well-being

    McClelland's 3 Needs: Achievement · Power · Affiliation
    Forms of Power (PM)
    Reward · Expert · Legitimate (Formal) · Referent (respect/personality) · Punishment (least desirable)
    📌 Exam Tip
    Best forms of power: Expert & Reward. Worst: Punishment. Performing is the stage where teams actually DO the work. Adjourning = team release/disbanding.
    ⚖️
    Manage Team
    Executing · Resource
    ▾
  • Tracking performance, providing feedback, resolving issues, managing team changes
  • Requires communication, conflict management, negotiation, and leadership
    • Conflict Resolution Methods (Best to Worst)
    Problem Solving/Confronting — Win-Win (BEST)
    Collaborating — find common ground
    Compromising — Lose-Lose (neither fully satisfied)
    Forcing — Win-Lose (one side dominates)
    Smoothing/Accommodating — temporary, Lose-Lose
    Withdrawal/Avoidance — Yield-Lose, conflict NOT resolved (WORST)
    Emotional Intelligence (EQ)
    Managing personal emotions of oneself and others. Key PM soft skill.
    📌 Exam Tip
    Greatest project conflict: between PM and Functional Manager (over schedules, priorities, resources). Problem Solving = BEST conflict resolution. Withdrawal = WORST.

    Monitor and Control Project Work
    M&C · Integration
    ▾
  • Tracking, reviewing, and recording progress to meet PM Plan performance objectives
  • Takes Work Performance Information → creates Work Performance Reports
    • Data Flow
    Work Performance Data (raw, from Executing)
    → Work Performance Information (analyzed/compared)
    → Work Performance Reports (distributed to stakeholders)
    🔄
    Perform Integrated Change Control
    M&C · Integration
    ▾
  • Reviewing all change requests; approving/rejecting changes to deliverables, documents, PM Plan
  • Any stakeholder may request a change (must be in writing)
  • PM assesses impact; submits to Change Control Board (CCB)
  • CCB — responsible for reviewing, evaluating, approving, deferring, or rejecting changes
  • Output: Approved Change Requests → implemented in Direct & Manage Project Work
    • 📌 Exam Tip
    The PM does NOT approve changes alone — that's the CCB's role. Approved change requests are implemented in the Executing process group. Change Log tracks all decisions.
    ✔️
    Validate Scope
    M&C · Scope
    ▾
  • Formalizing acceptance of completed project deliverables
  • Verified deliverables reviewed with customer/sponsor for formal acceptance
  • Done at the same time as or immediately after Control Quality
  • Concerned with correctness of the deliverable
  • Output: Accepted Deliverables — signed off by sponsor/customer
    • 📌 Exam Tip
    Quality Control verifies correctness (internal). Validate Scope gets formal customer acceptance (external). Validate Scope uses VERIFIED deliverables from Control Quality as input.
    🔭
    Control Scope
    M&C · Scope
    ▾
  • Monitoring status of project/product scope and managing changes to scope baseline
  • Scope Creep — uncontrolled expansion WITHOUT adjustments to time, cost, resources
  • When changes approved, project baselines must be adjusted
  • Tools: Variance Analysis + Trend Analysis
    • ⏰
    Control Schedule
    M&C · Schedule
    ▾
  • Monitoring status to update schedule and manage changes to schedule baseline
  • Compare work results to plan — identify variances
  • Tools: Earned Value Analysis, Performance Reviews, What-If Scenarios, CPM, Resource Optimization, Schedule Compression (Fast Track/Crash)
  • Output: Schedule Forecasts — based on past and expected future performance
    • 💵
    Control Costs
    M&C · Cost
    ▾
  • Monitoring status to update costs and manage changes to cost baseline
  • Primarily concerned with cost variance
  • Any budget increase must go through Perform Integrated Change Control
  • Output: Cost Forecasts — uses EAC (Estimate at Completion) EVM formula
    • 📌 Exam Tip
    EVM formulas are major exam topics — see the Formulas tab for all EVM calculations. TCPI (To-Complete Performance Index) tells you how efficiently you need to work to meet budget goals.
    🔍
    Control Quality
    M&C · Quality
    ▾
  • Assessing performance — project outputs complete, correct, meet expectations
  • Each deliverable inspected, measured, and tested
  • Data Gathering: Checklists · Check Sheets · Statistical Sampling · Questionnaires
  • Output: Verified Deliverables (input to Validate Scope)
  • Control Charts — "Rule of 7": 7 consecutive points on one side = out of control

    Formulas, Mnemonics & Exam Tips
    All key calculations and memory aids for the PMP exam in one place.
    📐
    PERT / Three-Point Estimating
    Schedule & Cost Estimating
    ▾
    Beta Distribution (PERT Weighted Average)
    E = (O + 4M + P) / 6

    O = Optimistic · M = Most Likely · P = Pessimistic
    Triangle Distribution
    E = (O + M + P) / 3
    Standard Deviation
    SD = (P − O) / 6
    Example
    O=8, P=14, M=10
    Beta: (8 + 40 + 14) / 6 = 62/6 = 10.33 days
    Triangle: (8 + 10 + 14) / 3 = 10.67 days
    SD: (14−8)/6 = 1.0 day
    📡
    Communications Channels Formula
    Plan Communications
    ▾
    Channels Formula
    Channels = n(n−1) / 2

    n = number of people on the project
    Examples
    4 people: 4(3)/2 = 6 channels
    10 people: 10(9)/2 = 45 channels
    11 people: 11(10)/2 = 55 channels (+10 from adding 1 person)
    📌 Exam Tip
    Adding 1 person to a project adds n new channels (where n = old team size). This shows why communication complexity grows rapidly with team size.
    💹
    Expected Monetary Value (EMV)
    Quantitative Risk Analysis
    ▾
    EMV Formula
    EMV = Probability × Impact (cost)
    Total = Initial Cost + EMV
    Decision Tree Example
    Option A: $1M initial + $400K risk @ 25% → EMV = $100K → Total = $1.1M
    Option B: $800K initial + $500K risk @ 10% → EMV = $50K → Total = $850K
    → Choose Option B (lower total)
    📌 Exam Tip
    Negative EMV = threat (cost). Positive EMV = opportunity (benefit). Use Decision Tree Analysis to compare alternatives. Choose lowest total cost (or highest benefit for opportunities).
    💰
    Cost Estimate Accuracy Ranges
    Estimate Costs
    ▾
    Estimate Types
    Rough Order of Magnitude (ROM): −25% to +75%
    (Early project, limited info)

    Budget Estimate: −10% to +25%
    (Mid-project, more planning done)

    Definitive Estimate: −5% to +10%
    (Late project, detailed scope known)
    📌 Exam Tip
    Accuracy IMPROVES as the project progresses. Early = ROM (widest range). Later = Definitive (most accurate). The negative range is always smaller than the positive range.

    Contract Type — Risk Summary
    Plan Procurement
    ▾
    Risk Distribution
    ← More Buyer Risk ________________________ More Seller Risk →

    T&M      Cost-Reimbursable      Fixed Price

    Fixed Price: Seller bears cost overrun risk. Use when scope is well-defined.
    Cost-Reimbursable: Buyer bears cost risk. Use when scope is unclear.
    Time & Material: Buyer takes all risk on both labor and materials.
    📌 Exam Tip
    If an exam scenario says "scope is unclear" → T&M or CR contract. "Scope is well-defined" → Fixed Price. FFP = no changes allowed. FPIF = performance incentives built in.
